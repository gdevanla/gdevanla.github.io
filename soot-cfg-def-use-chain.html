Using Soot to obtain def-use chains - A Simple Example

08 Mar 2014 In programming, static analysis, research.

Soot provides very powerful framework to perform different kinds of analysis on JVM byte code. This post has a simple example on how to use Soot to obtain def-use chains in a given method. You need to be familiar with Soot in general and some terms of static analysis to follow this example.

Here is the simple example

UnitGraph unitGraph  = new EnhancedUnitGraph(body)
SimpleLocalDefs simpleLocalDefs = new SimpleLocalDefs(unitGraph)

//now say `u` is a variable pointing to a unit
//and `l` is one of the locals used in `u`.
//Then we can use the following statement to obtain
//all defs of `l` reachable from `u`
List<Unit> defUnits = simpleLocalDefs.getDefsOfAt(l, u)

//note that a local `l` at `u` could have more than one
//def since the different def statements could be reached
//based on conditional constructs.

We add this code in the internalTransform() function.

First step is to build a UnitGraph. There are different ways to build this object. Depending on how much details the graph needs to include.

UnitGraph unitGraph = new EnhancedUnitGraph(body)

Other version of UnitGraph would be BriefUnitGraph, EnhancedUnitGraph, ExceptionalUnitGraph and TrapUnitGraph.

Second step is to build the SimpleLocalDefs object.

And finally, we can use simpleLocalDefs.getDefsOfAt(Local, Unit)to gets all reaching defintions of locallat unitu. Other option would be to useSmartLocalDefs`. But, I believe it is computationally more expensive in how analysis is performed.
