How to build and deploy an Scotty-based Haskell webapp to Elastic Beanstalk

09-07-2016 In Haskell, Web Development.

Steps required to build and deploy a Scotty-based web application

Getting the stack environment ready

 stack new helloscotty scotty-hello-world
 stack build

update template to use scotty >= 0.11

Binary file available at

    .stack-work/install/x86_64-linux/lts-5.18/7.10.3/bin/helloscotty

Amazon Elastic Beanstalk does not have a way to directly deploy Haskell based web apps.

So, we use the docker container to deploy our web app

docker:
    enable: true
    repo: fpco/stack-build

    stack docker pull

$ stack docker pull
Pulling image from registry: 'fpco/stack-build:lts-7.14'
lts-7.14: Pulling from fpco/stack-build

Digest: sha256:562d2fcb1c08f893700a2445b96c1310b8b8836edf083d82982d39caa310cdc8
Status: Downloaded newer image for fpco/stack-build:lts-7.14

stack build

stack --docker-run-args='--net=bridge --publish=3000:3000' exec helloscotty

This ensures your web app build successfully with the new docker image. In addition, we will need the executable the build generated.

Create docker file

Publish to docker hub

Follow instructions to set up Elastic Bean stalk application

Creat docker environment

Follow instructions as provided in this blog post

http://thequietlattice.com/docker/aws/elasticbeanstalk/2015/12/18/deploy-private-docker-to-elb.html

Some notes on this blog:

  • Update container port 8888 to 3000
  • 6th step is quite different. After creating the environment, switch to configuration tab to set up the instance profile
