Parametric Types in Haskell

09-05-2016 In functional programming, Haskell, data-structures, design.

In Haskell, we can have what we call parametric functions, that are functions that takes parameters that are generalized to a family of types (instances of a set of type classes). Additionally, we can also have data constructors that can take be defined with parameterized types.

Some examples:

Here we define a parameterized record type.

1
2
    data Instrument a = Instrument {instrumentName::String
                                    , instrumentLevel::a}

The parameterized record type can be used in function's that are also parameterized.

1
2
    useGenericInstrument :: Instrument a -> String
    useGenericInstrument i = instrumentName i

We could constrain the functions to only take, certain instances of parameterized types,

1
2
    useGenericInstrumentsParametricType :: (Num a, Eq a) => Instrument a -> a
    useGenericInstrumentsParametricType i = instrumentLevel i + 10

Or, define functions for specific types

1
2
    useSpecificInstrumentType :: Instrument Int -> Int
    useSpecificInstrumentType i = instrumentLevel i + 10

We can also use ad-hoc polymorphism(link here), to specify overloaded functions for different types on Instrument.

1
2
3
4
5
6
7
8
    class MethodsThatWorkWithInstruments a where
      useLevel:: a -> Int

    instance MethodsThatWorkWithInstruments (Instrument Int) where
      useLevel i = instrumentLevel i + 10

    instance MethodsThatWorkWithInstruments (Instrument String) where
      useLevel i = (read $ (instrumentLevel i) :: Int) + 10

Nested Parametric Types

Parametric types can be nested into other types. For example,

1
2
3
4
5
6
    data InnerRecord a = InnerRecord {innerRecordName::String
                                      , innerRecordType::a} deriving (Show)

    data OuterRecord a = OuterRecord {
          outerRecordName::String,
            outerRecordInnerRecord::InnerRecord a}

It is important to note that, nesting parametric types could get cubmersome, if we have multiple levels of wrapping around parameterized types. The type variable a will have to be passed all the way from the outermost declaration. For example,

1
2
3
4
5
   --- type variables can be generic all the way from the top
   data OuterRecords a = [OuterRecord a]

   -- or specialized at some top level
   data OuterIntRecords = [OuterRecord int]

We probably want to specialize the type as soon as we can at the lower level. We could do this in 2 ways,

1
2
3
4
5
    data InnerRecordType = Good | Bad

    data OuterRecord = OuterRecord {
         outerRecordName::String,
         outerRecordInnerRecord::InnerRecord InnerRecordType}

Or, if we want totally independent types,

1
2
3
4
    data Up
    data Down
    type GoodRecord = InnerRecord Up
    type BadRecord = InnerRecord Down

The former option is useful, if we have to create a list of OutermostRecords or group them together in some other ways.
