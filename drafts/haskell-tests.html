The unreasonable flexibility of creating tests in Haskell(using HUnit)

05-12-2017 In Haskell, testing.

One can use HUnit for writing test in Haskell. HUnit provides a very flexible way of creating tests and grouping them. Using HUnit, tests can be arranged in a tree like hierarchy irrespective of where they are defined. Each node in the test tree can either be a node that contains children, which are themselves group of tests or individual tests.

Each individual tests, is an instance of TestCase, that can be created using testCase is of type Assertion.

    type Assertion = IO ()

Therefore, to generate multiple testCase objects, that test a common invariant, all we need to do is call a function such as below for different inputs, and let the function return a TestCase instance.
