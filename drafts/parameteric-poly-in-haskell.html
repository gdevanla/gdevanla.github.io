Parametric Polymorphism in Haskell

09-05-2016 In functional programming, Haskell, data-structures, design.

In a previous post, we took a look at some examples on how type classes in Haskell can be used to implement constructs for ad-hoc polymorphism. In this post we will look at another kind of polymorphism, namely, parameteric polymorphism.

In ad-hoc polymorphism, we specialize functions to work on certain types. Using parametric polymorhism, we generalize functions to work on a family of types. Say, we have a function defintion,

    incompleteGenericFunction :: a -> Bool
    incompleteGenericFunction arg = doSomethingWithTypeA arg


    doSomethingWithTypeA :: Int -> Bool
    doSomethingWithTypeA arg = arg > 10

In the above code, we could attempt to define doSomethingwithtypeA method, along with the function incompleteGenericFunction that takes a generic type a. But, we just gave the compiler conflicting instructions. First we suggested, the doSomethingWithTypeA function only accepts an Int, but we declared that incompletegenericfunction is accepting a more generic type. Here is the error, we will see if we attemt to compile the above code.

    Couldn't match expected type ‘Int’ with actual type ‘a’
           ‘a’ is a rigid type variable bound by
                     the type signature for incompleteGenericFunction :: a
           -> Bool
                     at app/Polymorphism.hs:10:30

Here, we have to prove to the compiler that any type a that will be passed to inCompleteGenericfunction will have a corresponding function doSomethingWithTypeA that accepts that type. Now, how could we possibly do that. The possible types for a is not finite. We can sort this out, by using type classes. We can be explicit and inform the compiler that, a function taking a generic type, will accept a generic type provided it provides an instance of a particular or set of type classes. For example,

    class ParametricPolyExample a where
      doSomethingWithTypeA:: a -> Bool

    instance ParametricPolyExample Int where
      doSomethingWithTypeA x = x > 10

    instance ParametricPolyExample String where
      doSomethingWithTypeA x = len x > 10

And, then we can use this information in our generic function, as

    --Example of parametric polymorphism
    aGenericFunction :: (ParametricPolyExample a) => a -> Bool
    aGenericFunction arg = doSomethingWithTypeA arg

Here we explicity promised the compiler that our aGenericFunction , for all types a, this function will be called with there, that type a will be an instance of ParametricPolyexample.

Therefore, we were able to make our function arguments generic upto one or more type families.

So far, we extended our examples from ad-hoc polymorphism to parametric polymorphism in terms of functions for simple types. These same examples can be extended to paramteric types as well.

Here is an example,

    data Student a = Student {name::String, level::a}

    class StudentTypeClass a where
      useStudent :: a -> String

    instance StudentTypeClass (Student Int) where
      useStudent s = "using student as Int"

    instance StudentTypeClass (Student String) where
      useStudent s = "using student as String"


   -- we can also have functions, that work only on specific Student
   -- type
      useStudentWithLevelAsString :: Student String -> String
      useStudentWithLevelAsString s = level s

In the above example, we have useStudent implementing version for types of of levels, whereas the useStudentWithLevelAsString is specific to String type of level.
